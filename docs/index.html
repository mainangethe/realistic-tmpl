Introduction

Thank you for getting this website template from our company. This manual will show you how to work with your template, how to edit it and make it a "live" website.

All pages content is made with HTML 4.

Unzipping the Template Package

After unzipping the template package you will find 5 folders:

  • css
  • docs
  • images
  • js
  • pages

General Information

Software Requirements

  1. A Text Editor/IDE. Used to edit the .HTML, .JS & .CSS files.
  2. A Browser. Used to view the changes in your computer prior to deploying.

The following scheme displays the templates files structure.

  • css - contains all the .css files used in the main template.
    • forms.css - contains all the styles for form elements used.
    • gallery.css - contains all the styles for gallery page.
    • homepage.css - contains all the styles for homepage elements.
    • navigation.css - contains all the styles for all navigation elements present.
    • portfolio.css - contains all the styles for portfolio page.
    • style.css - the main css file for the theme.
    • tables.css - contains all the styles for table elements used.
  • docs - contains all the files used in this documentation.
  • images - contains all the images used in this template.
    • icons - contains all the icons used.
      • black_file.gif - used in navigation elements
      • hpage_latest_shadow.gif - used in homepage-latest project section.
    • gallery-1.jpg - used to showcase the template's lightbox.
    • screenshot-index.png - used in the README.
  • js - contains all the jQuery plugins used.
    • fancybox - contains the lightbox jQuery plugin files.
    • superfish - contains the menu-builder jQuery plugin files.
  • pages - contains all the pages in the theme.
    • full-width.html - page with a layout where all elements utilize the whole browser's width
    • gallery.html - page with images linked to a lightbox.
    • portfolio - page with case study showcases.
    • style.demo - page showcasing other elements available for use.
  • index.html - homepage of the template.
  • README.md - page containing template instructions.

Editing the Template

Working with the template you need to edit the .html & .css files.

You can open any HTML with your preferred text-editor/IDE.

CSS files are used to change the template's appearance. Options include fonts, backgrounds, images, column-width values etc.
You can edit the css files with your preferred text-editor/IDE.

Customization Tutorials

How to edit Text

This being a HTML based template means all content is stored and could be accessed through the .html files.

  1. Open your template package
  2. Open index.html file with your text editor/IDE.
  3. You can use the search tool CTRL+F to find any text you need to change.
  4. Edit the text to your preferred choice and save the file.
  5. Open the file with your browser and you should see the changes reflecting.

There is another option that involves the browser. You can use the browser's developer tool to find the blocks of text you want to edit. You can learn more about the developer tools checking the browser's tutorials.

How to edit images

Open the template package and go to the 'images' folder. Here you can upload your images with the same names and extension to replace the default ones.

If you don't want to change the names/extensions of your custom images. Then there's an extra step of changing the image also in the specific html/css file.

  1. Open the .html or .css site with your preferred text editor.
  2. Use the search tool CTRL+F to find the image within the file.
  3. Replace the image source and save the respective file.
  4. Open the browser to see the new changes.

Uploading the Template

In order to make your site "live", you need to upload all the content of the template from your local computer to your hosting server.

Please note: your website root depends on the directory structure of your hosting server

You can upload the files using free FTP software e.g. FileZilla or CyberDucky.

Others have webFTP e.g. cPanel(WebHost Manager WHM).

We strongly advise against uploading the source directory or any of the template's source files to the hosting server.
